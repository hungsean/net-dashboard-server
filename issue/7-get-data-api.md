# 7 - get data api（GET API 從 D1 回傳數據給 web）

## 背景 / 為什麼

web 端需要一支 GET API 取得 dashboard 數據。
資料已清洗並存入 D1（#6），本棒做出**從 D1 讀取並回傳**的 GET API，並依人要求**加上驗證**（#1）。

## 背景前提
本棒須在 **#6（資料已進 D1）** 完成後才有資料可回。

## 範圍

### 要做什麼
- 新增一支 GET 路由（例如 `GET /data`），掛上 #1 的 auth middleware。
- 從 D1 讀取數據（依 #4/#6 的結構），轉成 web 端好用的 JSON 回傳。
- 基本查詢能力：至少能取得目前數據（如需分頁 / 篩選，於實作時依 web 需求最小實作或記錄為後續）。
- 無資料時回合理的空結果（非錯誤）。
- 錯誤處理：D1 讀取失敗回 5xx 並有清楚訊息。

### 不做什麼
- 不做寫入 / 上傳（那是 #2、#3、#5、#6）。
- 不實作 auth 本身（用 #1）。
- 不做複雜的聚合 / 報表運算（如有需求另開 Issue）。
- 不改資料層設計（用 #4/#6）。

## 驗收條件
- [ ] 有一支受 auth 保護的 GET 路由。
- [ ] 帶正確 token → 從 D1 取得數據並回傳結構化 JSON。
- [ ] 無 / 錯 token → 被 #1 middleware 擋下（401/403）。
- [ ] D1 無資料時回合理空結果，不報錯。
- [ ] D1 讀取失敗時回 5xx 與清楚訊息。
- [ ] 端到端：上傳→清洗→存 D1→本 API 取得，可拿到剛上傳的數據。

## 預估大小
小～中（單一讀取路由 + 序列化）。

## 相依關係
- **依賴 #6**（D1 已有資料）與 **#1**（auth）。
- 為整條流程的最後一棒。
